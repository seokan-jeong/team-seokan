# Debate Skill

여러 전문가 에이전트가 토론하여 최선의 결과를 도출합니다.

## 트리거

- "토론해줘", "debate"
- "의견 모아줘", "논의해줘"
- "어떤 방법이 나을까", "장단점 비교"

## 토론 패턴

### 1. 라운드 테이블 (Round Table)
모든 참여자가 순차적으로 의견을 제시하고 상호 피드백

### 2. 변증법 (Dialectic)
정(Thesis) ↔ 반(Antithesis) → 합(Synthesis)

### 3. 전문가 패널 (Expert Panel)
각 도메인 전문가가 자신의 관점에서 의견 제시

## 토론 프로세스

```
┌─────────────────────────────────────────┐
│ Phase 1: 문제 정의 & 패널 소집          │
│ 이슬(Moderator)가 주제 정의, 참여자 선정 │
└─────────────────────┬───────────────────┘
                      ↓
┌─────────────────────────────────────────┐
│ Phase 2: 의견 수집 (병렬 실행)          │
│ 각 에이전트가 독립적으로 의견 제시       │
│ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐       │
│ │ A   │ │ B   │ │ C   │ │ D   │       │
│ └─────┘ └─────┘ └─────┘ └─────┘       │
└─────────────────────┬───────────────────┘
                      ↓
┌─────────────────────────────────────────┐
│ Phase 3: 토론 라운드 (최대 3회)         │
│ 상호 피드백 및 논점 정리                 │
└─────────────────────┬───────────────────┘
                      ↓
┌─────────────────────────────────────────┐
│ Phase 4: 합의 도출                       │
│ 신형만(Oracle)이 의견 종합하여 최종안    │
└─────────────────────┬───────────────────┘
                      ↓
┌─────────────────────────────────────────┐
│ Phase 5: 검증                            │
│ 액션가면(Reviewer)이 합의안 검토         │
└─────────────────────────────────────────┘
```

## 토론 규칙

1. **최대 라운드**: 3회 (무한 논쟁 방지)
2. **발언 제한**: 각 에이전트 최대 500토큰
3. **합의 실패 시**: 투표로 결정 또는 에스컬레이션
4. **중재자 개입**: 논쟁이 평행선일 때 이슬이 중재

## 참여 에이전트 자동 선정

| 주제 키워드 | 소집되는 에이전트 |
|-------------|-------------------|
| UI, 프론트엔드, 컴포넌트 | 수지, 신형만 |
| API, 백엔드, DB | 흑곰, 신형만 |
| 배포, 인프라, DevOps | 훈이, 신형만 |
| 아키텍처, 설계 | 신형만, 유리, 봉미선 |
| 전체 시스템 | 수지, 흑곰, 훈이, 신형만 |

## 출력 형식

```markdown
## 토론 결과

### 주제
[토론 주제]

### 참여자
- 에이전트A (역할)
- 에이전트B (역할)

### 의견 요약
| 에이전트 | 핵심 의견 | 장점 | 단점 |
|----------|-----------|------|------|
| A | ... | ... | ... |
| B | ... | ... | ... |

### 토론 하이라이트
- Round 1: ...
- Round 2: ...

### 합의안
[최종 합의된 방안]

### 근거
- 이유 1
- 이유 2

### 검증 결과
✅ 승인 / ❌ 수정 필요
```
